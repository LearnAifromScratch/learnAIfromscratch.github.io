Curriculum

I split this up depending what you're interested in from easiest to get into to hardest

  • Threat Analyst
    • We analyze job listings for this and see how fast we can teach ourselves the requirements because these are great side incomes investigating logs looking for problems.
  • Pen testing/AppSec
    • The company that sells Burpsuite maintains one of the best security blogs, if it's important to know it'll probably be there. They also have an academy of labs we'll do.
    • MIT's 6.858 Computer Systems Security course from 2020 is open w/lectures on YouTube, and the material is not very different from 2022. More importantly all the labs are open.
  • Exploits for fun and profit
    • Brown's exploit course is mostly open (no lectures, but assignments are open), and their grad course which is mainly papers about memory corruption Return-Oriented Programming techniques like JIT-ROP which will not go away ever with current architecture and still works. We'll also do MIT's CPU hacking course learning about side-channels, cache-attacks, the 'trusted platform' attacks and rowhammer or flipping bits in memory without even accessing them.
  • Post-Quantum Crypto/Cryptanalysis
    • Done in this workshop though we'll see some of it here. This is the most critical security because all your data is being vacuumed up and sitting somewhere waiting to be decrypted once 'quantum supremacy' happens.

PreReqs: How bad it all is

If you haven't seen it, watch the CS19 lecture on security. I assume you did that whole course.

Immediately npm breaches is brought up, the JavaScript package manager. You may have tried to install some Node.js software to run it locally and noticed there's hundreds of dependencies being automatically installed. If you missed the attack description the stream dependency was attacked to specifically target some Bitcoin wallet software in hopes that the wallet developers themselves would include it in their builds and they did. Hilariously all the attacker had to do was find some unmaintained but popular dependency and took over the entire project. Many browser plugins have had this happen too, either sold to adtech or malware writers or they helpfully take over the abandoned project. Shady adtech has done this too they buy QR code reading software in Google Play Store and 'enable ads' meaning when you scan a QR code, it fools the user into handing over financial details to some scam subscription.

@9:00 we're getting into nightmare scenario where VSCode the editor most developers use is now compromised. The point is some dependency nobody pays attention to got turned into malicious code and found it's way into every critical program on earth almost.

@19:50 the seemingly benign software you find in the wild like adding colors to your terminal or javascript console is actually filled with malware of course but you probably knew that already. Famously this was almost all the flashlight/torch apps on app store before they cracked down.

@28:14 he's reminding us again the most dangerous software is the software you least expect to be dangerous. @35:27 every basic shell copy program in every OS will bring in fopen() and then you're screwed, it has access to the entire filesystem. Where can you learn bourne shell > or < read, either MIT's course on the Missing Semester of your CS Education or the book The Unix Programming Environment by Pike & Kernighan from the 1980s which is a tiny book and well worth reading (we'll do it after this). I once used that book to pass a resume filter for StackOverflow you had to solve some puzzle in the shell and IIRC it was on like the 20th page or so of that book.

'The internet, operating systems, all programming languages, these things were built to be maximal insecure'.

He talks about SELinux also known as MAC or mandatory access controls. MAC is something security people love because it's knob tweaking but states become so complex in modern systems you cannot guarantee your policy doesn't allow illicit states without some kind of logical assistant like Alloy/Forge or other SAT prover. SAT is a logical connectives problem like p1 or p2 or p3 and what combination of T/F satisfies that problem, they are known NP-Complete problems so MAC quickly gets out of control if you go overboard.
