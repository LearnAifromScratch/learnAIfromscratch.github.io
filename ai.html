Reading: IMLP Ch 1, APM Ch 1-2 Both books can be found on libgen, reading APM 1.3 Terminology is helpful for understanding 'class' or 'predictors'. Chapter 2 (and the first lecture) introduces a linear and quadratic (power) model. These are both covered in MMAC in the first chapter. The first is a linear function f(x) = mx + b (well, technically affine function) who's parameters are m the slope (rise/run) and b the vertical y-intercept. Now you know what 'parameters' means in the context of modeling, in this case it is the 2 things that define what a linear line is. The second is a quadratic f(x) = ax^2 + bx + c which is really a power function model y = Cx^k where C and k are nonzero constants: f(x) = Cx^2 + Cx^1 + c. The parameters C and k define what it means to be a power function. Note setting parameter k to k - 1 in f(x) = ax^2 + bx + c gives you a linear function f(x) = ax + (b + c) or mx + b so a linear function is a special case of a quadratic function which itself is just a power function model for our purposes of machine learning.

Let's read his book, IMLP. In the lecture he also said What question(s) am I trying to answer? Do I think the data collected can answer that question? as being fundamental. We are taught the fundamental NumPy data structure the NumPy array and the comma denotes a second dimension in np.array([1,2,3],[4,5,6]). In a matrix/vector it's dimensions is it's length. There's a mention here about efficient sparse matrices handling with SciPy which is explained here basically they come up all the time and multiplying matrices together is an inefficient problem unless of course they are sparse then you can speed this up with SciPy built-ins. We will learn what a matrix is in the linear algebra crash course we do. The matplotlib section is the subject of the entire next lecture.

The first application (page 20) is very similar to the petal length k-nearest neighbors example in Kevin Murphy's book, which we'll also read in CS4780. Terminology is clearly defined here. Theory 2

If you forgot sets/function notation read this.

\[L_{D,f}(h) = D[{x: h(x) \neq f(x)}]\]

He changes this later, Empirical Risk Minimization notation here at the beginning of the lecture means a loss function L, over the set D and a function f. It consumes a function h as input then outputs a new distribution where h(x) labels do not match f(x) labels. D is the 'set of all papayas in the world' essentially it means the distribution where we source papayas that we don't know anything about. Function f is a hypothetical perfect labeller, basically your brain looking at a feature set and knowing how to label each input. D[{x: h(x) $≠ f(x)}] is the new distribution created of misses. He redefines this function later in the lecture.

\[L_{S}(h) = \frac{|{i: h(x_{i}) \neq y_{i}}|}{|S|}\]

The Empirical Loss notation \(L_{S}(h)\) is another loss function over the set S of samples, consumes a function h for input, and returns a ratio of the size of the set of mismatches over the size of all the samples S.

@17:20 the reason why the error is 0, is because h(x) predictor function is a simple if-statement that checks if x is already in the set S of samples and returns y label, N otherwise. Of course there is no loss because we have 'overfitted' our model, meaning it will work only for input data exactly the same as our training/sample data. Using his empirical loss function over the set S, there will be no errors where h(x) new input doesn't equal y label in set S, as the unknown inputs we labelled N don't exist in the set S and this function will only return the set of mislabelled inputs, not one's who don't have a label at all.

\[ERM_{H}(s \in S) \in {ArgMin(L_{S}(h \in H))}\]

New Empirical Risk Minimization: pick predictor h() from your set of all predictors H with the lowest empirical loss.

Theorem: Let X be any set, Y the label set {0,1} and let H be a finite set of predictor functions h() from set X to Y. Assume training samples in set S is generated by some probability distribution D over the set X and labelled by some h() in H. Also assume the elements of S are picked i.i.d independent and identically distributed, so randomly choosing one sample can't help you learn another. Then \(ERM_{H}\) is guaranteed to come up with an h() that has small true loss given sufficiently large samples S.

Linear algebra

Recall polynomials from highschool, they are a specification of a function. Polynomials are a way to understand the properties that function has such as for which x is p(x) = 0, what is the maximum or minimum values of p(x), what is it's graph if we use polynomials to specify it?. Knowing these things tells you all you want to know about a function f(x) by specifying it with p(x) and taking it's statistics.

Linear algebra is primarily concerned with transformations which is any operation that transforms an input to an output. Linear means however you change the input causes the same changes in the output like doubling the input causes the output to be doubled. So we have this space, which is kind of like a transformation test bench to manipulate and find the statistics of objects in them much like polynomials but we have many more operations available in this space than regular algebra like transpose, a linear algebra operation that takes a column and turns it into a row or a row into a column. A matrix like polynomials can specify a function, you input a vector and it outputs a vector. This means a matrix can be abstracted to more than just a table of numbers and you can use linear algebra wherever there is a vector space and that includes all kinds of different mathematics, it's almost like the underlying theory of it all and for us the most critical subject in machine/deep learning so we'll have to put in work understanding it, but remember all we're doing is algebra with a few more features and everything is linear.

Basics of Linear ALgebra

Obtain Terence Tao's free notes on linear algebra. Do a casual read of the first few pages where he precisely defines transformations (any operation that transforms an input to an output), what linear means, what algebra is, how you are really just doing algebra except with additional operations like span, transpose, trace etc.

Scalars defined on page 6, any quantity which can be described by a single number. He notes how abstract this can get like complex numbers, volume, time, density. As long as your scalars obey the rules of algebra where x + y = y + x and a(b + c) = ab + ac then you can use it as a scalar. Ask yourself questions to better understand his definition: can n^2 be a scalar where n is some natural number? n^2 is a function which outputs a single number, sure. Test to see if it passes his scalar rules of algebra test.

Vectors defined on page 7 and axioms on page 8, a vector space is any class of objects which can be added together, or multiplied with scalars. The vector (1, 2, 3) can't be added to (5, 6) because addition is elementwise, you need equal length vectors to do addition. We already saw a vector in the first theory lecture, where papayas were described using an x and y 2D plane where the coordinates represented (color, hardness). If you wanted to add on additional features such as (color, mushy, price, organic) that's what Tao means by a vector in R^4 or R^n the more things you need to describe an object in that vector space, the bigger it's dimension. Page 10 an n-tuple is a math data structure, brackets with elements seperated by commas where the order matters. Tao notes R^1 the field of all real number scalars is itself a vector space inside a line. R^0 is a vector space inside a point. There's many examples here how so long as the space satisfies the axioms to be a vector space, then you can have vectors in it and do linear algebra. Notice sqrt(x) is not a vector in the polynomial vector space because if x is negative it's not defined in R, so you break out of the space (into the complex numbers), it's not closed under addition or scalar multiplication. A polynomial can't have a negative exponent either since no polynomial can contain division by a variable. He shows that even a matrix is a vector, the point being a very complicated object can be a vector and then be manipulated in a vector space.

Vector subspace defined on page 18, if you take a vector space and extract a part of it like a subset, it's only a subspace if addition and scalar multiplication is closed within that subset and doesn't escape to another vector space. The interesting trick here is proving vector space axioms III (additive identity) and IV (additive inverse) exist within an arbitrary subspace by simply multiplying any vector by 0 or -1 creating a zero vector and an inverse vector. Many examples of what a subspace can be follows, like if you zero out the z in (x, y, z) a 3 dimensional vector or R^3 vector, then you have created an R^2 vector subspace since addition and scaling cannot escape the subspace. Tao remarks that these subspaces tend to be lines or flat planes through the origin in R^2 and R^3.

That's good enough for now, we'll skip to the definition of matrices when it comes up.
