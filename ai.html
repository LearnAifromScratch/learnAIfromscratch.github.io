LLMs and Artificial General Intelligence

There is a disconnect between the blogger junk articles promoting ChatGPT and all these doomers running around claiming the singularity is upon us. What the blog junk and marketing doesn't tell you is that very Large Language Models have "emergent properties" which means they have abilities that they weren't programmed to do and there is no explanation yet for why they have these abilities (phase shift/physics maybe). Collectively a giga model can have a behavior but each component does not have this behavior.

Primer on Matrices

Skimming 5 pages of this pdf by Stephen Boyd it's all we need to start.

A matrix size is rows x columns and it can contain more then just numbers/scalars the entries can be cosine/sine or other functions. Basically a matrix is an encoding of a function that performs a linear transformation like rotations or reflections on the plane, shearing the plane moving everything above the x axis right and everything below left by some scale, stretching the plane, or taking something non-linear like a curve and transforming it via linear algebra to a linear approximation. That is essentially how vector calculus works it's a way of doing calculus in higher dimensions by transforming it all to a linear mapping because even though globally everything is non linear, locally (magnifying) it's linear. You can also project an n-dimensional plane onto a 3D or 2D plane you simply input a 3-dimensional vector (x,y,z) and the transformation matrix zeroes out the z producing (x,y).

Just like how function composition is associative so is multiplying matrices A(BC) = (AB)C and just like how function composition is not commutative neither is matrix multiplicaiton. A matrix multiplied by a vector is like giving a function an input, therefore the image of the matrix is the vector output.

It gets even more abstract. A 1x1 matrix is essentially a number. A 2x2 matrix is not only a transformation function it can also be a kind of generalized number with a richer arithmetic you can encode complex numbers in certain 2x2 diagonal matrices using scalars to multiply a matrix who's square is -1 thus a complex number.

Page 3 'basis vectors' using a representation of an arrow is best showcased in this vid where at the end he goes slowly up in dimensions. Page 4 a transpose is a way to take the dot product at a higher dimension (we'll see it in machine learning vids). A dot product tells you the cosign of two vectors (in euclidean space) and if you remember highschool trig if cosine is 0 then the angle is 90deg or 2 vectors are perpendicular if vec1 dot vec2 = 0. Page 5 you can stop reading after scalar multiplication.

Neural Networks

Watching Lecture 1 CS4780 by Kilian Weinberger probably the best instructor you will find for this material. The real lecture begins @32:27. He defines both y (labels or classification) and x (feature vector) after the intro. Notice the feature vector is cleaned/prepared data that somebody needs to figure out how to generate, this is almost never revealed by all the closed source LLMs. When you solve Google captcha 'choose all the bicycles in this picture' that's what you are doing preparing some kind of data for supervised learning. If you did cs19 in the other workshop you remember doing DocDiff the first assignment using bag of words vector representation.

Watching Lecture 2 CS4780 (skip to 5:20) tells you how to select the algorithm for the hypothesis function by running statistical loss functions and seeing which has the lowest errors, talks a bit how they test these hypothesis functions.

Watching Lecture 4 CS4780 (skip to 36:50) the Perceptron. This is an algorithm that models a single brain neuron and was the first machine learning algorithm. You get layers of perceptrons you have a neural network. We skipped K-Nearest Neighbors (it's worth watching if you've never seen it) it's quickly reviewed in lecture 5. The hyperplane notation is y = mx + b and familar linear notation for a straight line but it's a higher-dimensional generalization. Note he drew the w vector perpendicular to the plane and we know w^{T}x is a dot product to determine the cosine of 2 vectors and if it's 0 then the vectors are perpendicular. So for any vector x we can compute y = wx + b and if 0 = wx + b then x is on the hyperplane.
