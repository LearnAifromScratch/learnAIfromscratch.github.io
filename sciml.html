Extension install

Download the Julia language binary, expand it somewhere, set up your platform specific paths. Download VSCode which is free. Install the extension for Julia which consists of starting VSCode, File->Pref->Extensions and search for Julia in the extensions marketplace. Install it, go to File->Pref->Settings select extensions, Julia, and change the executable path to ~/julia-1.8.0-rc3/bin/julia if on linux or julia.exe if on windows, whatever the full file name is. Even if you made a system wide path sometimes VSCode can't find it. Find out the number of threads your CPU has (cores) and change the value in the extension, example linux: 

cat /proc/cpuinfo
or
grep 'cpu cores' /proc/cpuinfo | uniq

While in VSCode settings, click on Text Editor settings and disable every auto closing brackets, quotes, deletes or anything that is inserting/removing text automatically unless you want that. I also disabled the 'telemetry' we aren't power users so no reason to send usage data to the developers. This is all kept in ~/.config/Code/User/settings.json so you can edit that manually in the future. Read how intellisense works and how to configure it if you want.

Optimizing Serial Code

Watching Lecture 1 there's notes here. Arrays in Julia are indexed starting at 1 instead of 0. In the notes we see a 3x3 matrix where the interpretation of the program is laying columns out in a sequential array by Julia, so if you iterate by columns you get to take advantage of locality or a stride-1 pattern since the CPU has already lifted those nearby array elements into the cache expecting you to do something with them next. If you iterate by row in a 3x3 matrix you get stride-3 access meaning we are skipping every 3rd element and depending how big the array is this is called a cache miss where it has to fetch the data again and completely kills performance.

@7:47 if it's not obvious reshape(A,(3,3))[3,2] evaluates to a new array then [3,2] accesses it like A[3,2]. This is of course what he says is sugaring the syntax where A[3, 2] or A[row,column] really means A[row + (column-1)*(total columns)] or A[6] because of how the array is represented as a flat sequence of columns. He explains this again ~@58:20

The MIT course page includes these notes about SIMD hardware. Modern CPUs have special arithmetic registers that can hold multiple 64-bit integers/floats at the same time so 'vectorize' operations to be parallel and done elementwise like a vector operation in math where vector A + vector B is A1 + B1 + A2 + B2 ..

In Julia a fused loop (sometimes called loop jamming) is simply a map() applying a function (or nested functions) to each element yielding a new array. Finally his example plotting the output of mul! mutating linear algebra library function to multiply matrices vs the heap allocated version, this only matters up to about 100x100 matrices then the differences between functions doing n3 operations and n allocation shrink to where it doesn't matter, as a 1000x1000 matrix multiplication n3 operation will fully dominate any kind of one-time O(n) allocation which is why people still use Pytorch with massive matrices/vectors, otherwise they have to rewrite their loop code using numpy built-ins that feed results to a precompiled C function.

The alloctimer function uses @belapsed and according to the BenchMarkTools documentation @belapsed repeatedly runs the code to get an accurate benchmark returning the result in nanoseconds. This is why it takes so long to finish he loops through ns a vector with powers of 2 up to 27 and alloc-timer runs those multiple times. He plots this with a log-log plot so you can see it differing by a factor of 2 up until the last test of 128x128 size matrices.

Summary

  • Know how the language represents data in data structures (column ordered vs row ordered) and program to that model.
  • Give the compiler type information or use built-ins like StaticArrays.jl then it can predict what it's size will be and optimize.
  • Mutate existing arrays when storing the outputs of fused broadcasts.
  • Use macros/built-ins that return pointers to avoid new heap allocation.

Optimizing serial code 2

The second part of this topic is worth watching, he goes through all the LLVM IR code you don't have to know assembly. The @which macro is the best thing I've ever seen in a language where it tells you exactly where the definition is the source.

~28:00 this common lisp style of multiple dispatch is done for performance reasons due to the limiting nature of dynamic typing as there is only one single type a recursive sum type of all the multiple classes of values and case analysis is done at runtime to determine what class that value is and dispatch it to the proper function. An example of this would be how the complex numbers are defined. There is multiple classes of complex numbers: reals, integers, naturals, rationals, 'pure imaginary numbers', but in the set definition really everything is just a complex number so there is a single type C then a tag is added to all the other numbers to classify them. In dynamically typed languages there is a single type T made up of all class tags of functions and values.

If you're curious about the multiple dispatch/type annotations design decisions you may want to skim through Chapter 22 Dynamic Typing from Robert Harper's book PFPL. Chapter 22.1 Dynamically Typed PCF or DPCF is the core of Lisp but with variable binding fixed. There are only two classes of values: numeric and functions. The values are tagged with their class at run-time and these have to be checked every single time an operation is done on them, as there is no way to propagate the class of a value into a conditional branch, once inside the desired branch any operations there must be checked again.

Hence why Julia has multiple dispatch to avoid some of this performance penalty even though it's just a constant factor. The only static checking of DPCF and every other dynamically typed language is that there are no free variables in the expression, which is a variable that isn't bound to something. That's what syntax errors are, if you meant to type printf but typed pintf you have introduced a free variable pintf with no binding.
